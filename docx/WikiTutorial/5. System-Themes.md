# 🎨 System Themes

CarDeals provides three distinct visual experiences that can be switched at any time without reloading the page. Themes are powered by swappable external CSS files and managed globally through React Context.

---

## The Three Themes

### 1. Default (Light) Theme
**File:** `frontend/src/themes/theme-default.css`

A clean, professional look designed for everyday browsing. Uses neutral tones, crisp whites, and structured typography. Ideal for users who prefer a distraction-free interface focused on the vehicle listings.

---

### 2. Dark Theme
**File:** `frontend/src/themes/theme-dark.css`

A sleek, high-contrast dark background paired with luxury accents (gold or silver highlights). Uses advanced CSS transitions and layered `box-shadow` effects to give the platform an elevated, premium feel. Well-suited for low-light environments.

---

### 3. Sport Theme
**File:** `frontend/src/themes/theme-sport.css`

A bold, energetic color scheme using red and black (original Sport) or a brighter red-and-blue contrast variant. Designed to feel dynamic and sporty — matching the personality of performance vehicle listings.

---

## How Theme Switching Works

### Frontend Architecture

```
User clicks ThemeSwitcher.jsx
  → ThemeContext.jsx (updates active theme name in state)
  → App.jsx (reads active theme, applies the matching CSS file)
  → All components re-render with the new theme variables
```

| Component / File | Role |
|---|---|
| `ThemeSwitcher.jsx` | UI control — buttons or dropdown in the navigation bar for the user to pick a theme |
| `ThemeContext.jsx` | Global React Context — stores the active theme name and exposes a setter function |
| `useTheme.js` | Custom hook — shortcut for any component to read or change the current theme |
| `ThemeManager.jsx` | Admin-only page — lets administrators preview and set the default site-wide theme |
| `theme-default.css` | Stylesheet for the Light theme |
| `theme-dark.css` | Stylesheet for the Dark theme |
| `theme-sport.css` | Stylesheet for the Sport theme |

### How CSS Swapping Works
Each theme file defines the same set of CSS custom properties (variables) with different values. For example:

```css
/* theme-default.css */
:root {
  --bg-color: #ffffff;
  --text-color: #1a1a1a;
  --accent-color: #2563eb;
}

/* theme-dark.css */
:root {
  --bg-color: #0f0f0f;
  --text-color: #f0f0f0;
  --accent-color: #d4af37;
}

/* theme-sport.css */
:root {
  --bg-color: #111111;
  --text-color: #ffffff;
  --accent-color: #e63946;
}
```

All components across the app use these CSS variables for their colors. When the active theme CSS file changes, every element updates instantly — no page reload required.

## How to Switch Themes (Users)

1. Look for the **theme switcher control** in the top navigation bar (`Header.jsx`)
2. Click the `ThemeSwitcher.jsx` toggle — it shows the available theme options
3. Select your preferred theme; the entire site updates immediately
4. Your selection is stored in `ThemeContext.jsx` for the duration of your session

---

## How to Switch Themes (Admins)

Admins have access to the **Theme Manager** page at `/admin/theme` (`ThemeManager.jsx`):

1. Log in with an admin account
2. Navigate to **Admin → Theme Manager**
3. Preview each of the three themes in a live preview panel
4. Set the active default theme for all users visiting the site
5. Changes are applied via `ThemeContext.jsx` and propagated through `useTheme.js` to every component

---

## Adding a New Theme (For Developers)

To introduce a fourth theme:

1. Create a new CSS file in `frontend/src/themes/` — e.g. `theme-luxury.css`
2. Define all the same CSS custom property names (`:root { --bg-color: ...; }`) with your new values
3. Add the new theme name and file reference to `ThemeContext.jsx` and `ThemeSwitcher.jsx`
4. Add a preview card for it in `ThemeManager.jsx`

No other files need to be changed — all components already consume the CSS variables.
