# ЁЯЪА Installation Guide

This guide explains how to fully deploy the CarDeals platform тАФ including the MySQL database, Node.js backend, and React frontend тАФ on any server, including `myweb.cs.uwindsor.ca`.

---

## Prerequisites

Make sure the following are installed on your server before you begin:

| Tool | Minimum Version | Purpose |
|---|---|---|
| Node.js | v16+ | Runs the Express backend and React build |
| npm | v8+ | Installs frontend and backend dependencies |
| MySQL | v8+ | Hosts the cardeals database |
| Git | Any | Cloning the repository |

---

## Step 1 тАФ Set Up the Database

1. Log into your MySQL server:
   ```bash
   mysql -u root -p
   ```

2. Import the schema to create all tables (`users`, `cars`, `dealers`):
   ```bash
   mysql -u root -p < backend/database/schema.sql
   ```

3. (Optional) Seed the database with 20+ sample car listings, sample users, and dealership locations:
   ```bash
   mysql -u root -p < backend/database/seed.sql
   ```

   Alternatively, you can run the migration files individually in order:
   ```bash
   mysql -u root -p < backend/database/migrations/001_create_users.sql
   mysql -u root -p < backend/database/migrations/002_create_cars.sql
   mysql -u root -p < backend/database/migrations/003_create_dealers.sql
   ```

---

## Step 2 тАФ Configure the Backend Environment

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```

2. Copy or create the `.env` file and fill in your values:
   ```
   PORT=5000
   DB_HOST=localhost
   DB_USER=your_mysql_username
   DB_PASSWORD=your_mysql_password
   DB_NAME=cardeals_db
   JWT_SECRET=your_secret_key_here
   ```

   | Variable | Description |
      |---|---|
   | `PORT` | Port the Express server will listen on |
   | `DB_HOST` | MySQL server host (usually `localhost`) |
   | `DB_USER` | MySQL username |
   | `DB_PASSWORD` | MySQL password |
   | `DB_NAME` | Database name тАФ must match `schema.sql` (`cardeals_db`) |
   | `JWT_SECRET` | Secret key used to sign and verify JWT tokens |

---

## Step 3 тАФ Install & Start the Backend

1. Install Node dependencies:
   ```bash
   npm install
   ```

2. Start the backend server:
   ```bash
   # Production
   npm start

   # Development (with auto-reload)
   npm run dev
   ```

3. The Express API will start on the port defined in `.env` (default: `5000`).  
   Entry point: `server.js` тЖТ loads `config/app.js` тЖТ registers all routes.

   **Verify it's running** by visiting:
   ```
   http://localhost:5000/api/monitor/status
   ```
   You should receive a JSON response showing `db` and `api` as `"online"`.

---

## Step 4 тАФ Install & Start the Frontend

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```

2. Install React dependencies:
   ```bash
   npm install
   ```

3. Configure the API base URL. In `src/services/api.js`, ensure the `baseURL` points to your backend:
   ```js
   // Example for local development
   baseURL: 'http://localhost:5000/api'

   // Example for university server
   baseURL: 'http://myweb.cs.uwindsor.ca:5000/api'
   ```

4. Start the React development server:
   ```bash
   npm start
   ```
   The app will open at `http://localhost:3000`.

5. For a **production build** (static files to deploy):
   ```bash
   npm run build
   ```
   The compiled output will be in the `/build` folder. Serve it with any static file server or configure Express to serve it.

---

## Step 5 тАФ Deploying on `myweb.cs.uwindsor.ca`

1. Upload your project files to the server via SSH or SFTP
2. Import the database using the steps in Step 1
3. Set your `.env` values to match the university server's MySQL credentials
4. Run `npm install` and `npm start` in the `backend/` directory
5. Run `npm run build` in the `frontend/` directory and configure your server to serve the `/build` output
6. Make sure port `5000` (or your chosen `PORT`) is open and accessible

---

## Project Entry Points Summary

| Entry Point | File | Purpose |
|---|---|---|
| Backend server | `backend/server.js` | Starts Express, listens on PORT |
| Express app config | `backend/config/app.js` | Registers middleware and all routes |
| Database connection | `backend/config/db.js` | MySQL connection pool (mysql2) |
| React app | `frontend/src/index.js` | Mounts React into the HTML DOM |
| Root component | `frontend/src/App.jsx` | Wraps app in context providers, renders router |
| Route definitions | `frontend/src/routes/AppRouter.jsx` | Maps all URLs to page components |
