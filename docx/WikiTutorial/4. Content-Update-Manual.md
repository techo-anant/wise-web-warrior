# 🛠️ Content Update Manual

This guide is written for **non-programmers** and **site administrators** who need to update the CarDeals catalogue — adding new cars, swapping photos, updating videos, or editing dealership info — without touching any application code.

---

## 1. 🖼️ Updating Car Images

Car photos are stored in the frontend assets folder:

```
frontend/src/assets/images/cars/
```

**How to update:**
1. Prepare your new image file (JPG or PNG recommended, ideally 800×600px or larger)
2. Name it to match the existing file — for example, `toyota-camry-2023.jpg`
3. Replace (overwrite) the old file in `/assets/images/cars/` with the new one, keeping the exact same filename
4. The site will display the updated image automatically — no code changes needed

> **Tip:** Each car's image is linked via the `image_url` field in the database. As long as you keep the filename the same, the site picks up the new image instantly.

**Other image folders:**

| Folder | What It Contains |
|---|---|
| `/assets/images/banners/` | Hero section and promotional banner images (home page) |
| `/assets/images/logos/` | Brand logo and partner logos shown in the header/footer |

Replace files in these folders the same way — same filename, same folder.

---

## 2. 🎬 Updating Videos and Media

Car tour videos and audio files are stored here:

```
frontend/src/assets/videos/car-tours/
```

**How to update:**
1. Export your new video in MP4 format (recommended for broad browser support)
2. Name it to match the existing file for that car
3. Replace the file in `/assets/videos/car-tours/` keeping the exact same filename
4. The `VideoPlayer.jsx` component on each car's detail page will serve the updated video

> Car videos are linked via the `video_url` field in the database — same rule applies: keep the filename identical and no database changes are needed.

## 3. 🚗 Adding or Removing Vehicle Listings

Vehicle listings are stored in the **MySQL database**, not in a flat file. There are two ways to update them:

### Option A — Using the Admin Panel (Recommended)
1. Log into the site with an admin account
2. Go to **Admin → Car Listing Editor** (`/admin/cars`)
3. Use the form to **Add a New Car**, **Edit** an existing listing, or **Delete** one
4. Changes are saved to the database immediately and appear on the site in real time

Fields you can fill in:
| Field | Example |
|---|---|
| Make | Toyota |
| Model | Camry |
| Year | 2023 |
| Price | 28500.00 |
| Mileage | 15000 |
| Category | Sedan / SUV / Truck / Electric / Coupe / Van |
| Tag | New / Used / Featured |
| Color | Pearl White |
| Transmission | Automatic / Manual |
| Fuel Type | Gasoline / Diesel / Electric / Hybrid |
| Engine | 2.5L 4-Cylinder |
| Description | Full text description of the vehicle |
| Image URL | Path to the image file in `/assets/images/cars/` |
| Video URL | Path to the video file in `/assets/videos/car-tours/` |

### Option B — Editing the Database Directly
If you have database access, you can also insert or update rows directly in the `cars` table of `cardeals_db`. Refer to `backend/database/schema.sql` for the full table structure and `seed.sql` for example insert statements.

## 4. 🏢 Updating Dealership Information

Dealership locations, addresses, phone numbers, hours, and GPS coordinates are stored in the **`dealers` table** in the database.

**To update a dealership:**
1. Log in as an admin
2. Edit the dealer record directly in the database (admin UI for dealers can be added via `dealerRoutes.js` / `dealerController.js`)
3. Or update the `dealers` table directly in MySQL — fields include: `name`, `address`, `city`, `province`, `postal_code`, `phone`, `email`, `latitude`, `longitude`, `hours`

The **Dealer Locator map** (`DealershipMap.jsx`) reads from `GET /api/dealers` and will reflect any database changes automatically.

## 5. 🎨 Updating Promotional Banners

Banner images shown on the Home page hero section are stored in:

```
frontend/src/assets/images/banners/
```

Replace any banner image using the same filename as the existing one. The `Home.jsx` page references these images directly — no code change required when the filename stays the same.

---

## 6. 🔤 Updating Fonts

Custom fonts are stored in:

```
frontend/src/assets/fonts/
```

To add or replace a font:
1. Place the new font file (`.woff2` or `.woff` recommended) in this folder
2. A developer will need to update the `@font-face` declaration in `App.css` to reference the new filename — this is a one-line code change
